这**不是**一个组件库。它是一个可复用的组件集合，你可以将这些组件复制并粘贴到你的应用中。

**为什么说这不是一个组件库？**

我们的意思是，你不需要将其作为依赖项安装。它不通过 npm 提供或分发。

选择你需要的组件。将代码复制并粘贴到你的项目中，并根据你的需求进行自定义。代码是你的。

_你可以将其作为参考，构建你自己的组件库。_

## 常见问题

<Accordion type="multiple">

<AccordionItem value="faq-0">
<AccordionTrigger>

它是 shadcn/ui 的替代品吗？

</AccordionTrigger>
<AccordionContent>

不，它不是 [shadcn/ui](https://ui.shadcn.com/) 的替代品。
相反，Plate UI 是一个基于 [shadcn/ui](https://ui.shadcn.com/) 和 [Radix UI](https://www.radix-ui.com/) 构建的扩展组件集合，专门为**富文本编辑器**设计。对于应用的其他部分，你应该继续使用其他组件，例如 [shadcn/ui](https://ui.shadcn.com/)。

Plate UI 遵循与 [shadcn/ui](https://ui.shadcn.com/) 相同的约定、用法和模式，这意味着你可以无缝地将其与现有的 [shadcn/ui](https://ui.shadcn.com/) 组件集成，而不是替换它们。

在 [shadcn/ui](https://ui.shadcn.com/) 之后，Plate UI 是第一个发布的组件注册表。

</AccordionContent>

</AccordionItem>

<AccordionItem value="faq-1">
<AccordionTrigger>
为什么是复制/粘贴而不是打包为依赖项？
</AccordionTrigger>
<AccordionContent>
这样做的目的是让你拥有对代码的所有权和控制权，允许你决定组件的构建和样式。

从一些合理的默认值开始，然后根据你的需求自定义组件。

将组件打包为 npm 包的缺点之一是样式与实现耦合在一起。_组件的设计应该与其实现分离。_

</AccordionContent>
